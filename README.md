## 🤝 写作风格

我写技术博客的几个原则：

1. **真实场景驱动**：不写"理想架构应该是什么"，写"我们遇到了什么、怎么走过来的"
2. **数字优先**：用真实生产数据说话，避免"显著提升 / 大幅降低"等模糊词
3. **承认失败**：3 个失败方案 vs 1 个成功方案，失败的故事比成功有信息量
4. **代码可跑**：每篇博客如有代码，配套 GitHub repo 可 clone 验证
5. **不卖思路货**：不写"X 个 Y 的最佳实践"。要么深，要么不写
